# 偵錯工作階段
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

偵錯工具的架構的角度來看**工作階段**：  
  
-   是所有的設定，會先處理相關或無關的已啟動或連線到 IDE 的單一執行個體。  
  
-   工作階段偵錯管理員 \(SDM\) 來執行。  如需詳細資訊，請參閱 [偵錯的工作階段管理員](../../extensibility/debugger/session-debug-manager.md)。
