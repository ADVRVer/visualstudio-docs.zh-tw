# Visual Studio 的評估工具
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

## Visual Studio 而成的檢查清單  
 您可以使用這份檢查清單來評估使用者經驗的視覺與互動的詳細資料的品質。  
  
### 概觀  
  
-   確認所有命令都導致意見反應，告知使用者，其命令有尚未執行。  
  
-   確認所有 UI 項目和控制項都會顯示在所有主題與高對比模式。  
  
-   確認的作用與作用中的選取項目一律區分，同時以標準和高對比模式。  
  
-   請確認焦點永遠是可見，明顯。  
  
### 效能  
  
-   確認已顯示是否命令會使用超過 1 秒才能完成某些種類的 「 忙碌中 」 的指標。  
  
-   確認如果命令會使用 10 秒以上才能完成，明確進度列，請確定 \(慣用\) 或不定，隨即出現。  
  
### UI 文字  
  
-   確認所有的標籤是句子或字首大寫文字不是全部小寫。  
  
    ||更正|不正確|  
    |-|--------|---------|  
    |**命令文字 \(全部\)**|句子案例:<br /><br /> **目錄名稱:**|目錄名稱:|  
    |**按鈕文字 \(用戶端\)**|字首大寫:<br /><br /> **\[設為預設值\]**|設定設成 AS DEFAULT|  
    |**按鈕的文字 \(線上\)**|句子案例:<br /><br /> **\[設成預設值\]**||  
  
-   確認群組標頭和按鈕以外的所有標籤使用分號，並在前面的成對的控制項。  
  
-   確認以省略符號按鈕、 命令和命令連結來啟動 UI 以擷取使用者輸入結尾 **\[...\]**。  
  
     例如：  
  
    -   **\[進階...\]** 對話方塊上的按鈕。  
  
    -   在 \[工具\] 功能表下的命令選項 \(**工具 \> 選項**\) 應該取得省略符號，因為啟動對話方塊本身是命令的目的。  
  
-   確認 UI 中含有不縮寫，除了業界標準詞彙。 比方說，HTML 或 TCP\/IP 都不需要對拼出，不過應該 OOM \(記憶體不足\) 和 PII \(個人識別資訊\)。  
  
### 存取鍵盤  
  
-   請確認完成各項工作，使用鍵盤的方式。 通常這是透過鍵盤存取的每個控制項，但對某些高度視覺化的區域中，因應措施，例如程式碼\] 檢視是可接受。  
  
-   請確認您可以透過邏輯順序 \(由左到右及由上到下\) 中的控制項索引標籤。 雖然這是大多數控制項的最佳作法，並非所有控制項都需要這種方法。 例如，確認該控制項是一個定位之群組中的選項按鈕。  
  
-   確認所有控制項都有標籤，而且每個標籤有助憶鍵 \(例外狀況會包含一些非標示的控制項可能後面加上標籤的控制項索引標籤中\)。  
  
-   請確認沒有助憶鍵衝突。  
  
### 字型  
  
-   請確認所有字型 \(圖示、 大小、 色彩\) 一致的方式使用和維護階層。  
  
-   請確認所有 UI 項目都使用環境字型服務。 \(請參閱 [字型和格式適用於 Visual Studio](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md)\)  
  
     若要檢查服務是否正在使用，請移至 **工具 \> 選項 \> 字型和色彩**。 在 \[設定\] 下拉式清單中，選擇環境字型變更為 stylistically 不同 \(例如 Harrington 或漫畫 San\) 的字型和大小設定為 12 點。 然後按一下 \[確定\]。 您可能需要重新啟動 IDE 時，但是大部分的 UI 會立即變更。 不要拿字型變更，即使在重新啟動的區域並未使用環境字型。  
  
-   驗證服務 \(例如粗體或放大文字\) 的衍生的字型，保留其大小和環境字型的大小變更時，相對於 「 正常 」 的文字格式設定。  
  
-   確認沒有裁剪錯誤，因為放大字型。 被截斷的字型可能會將控制項固定的高度或固定的高度容器。  
  
### 對話方塊  
  
-   確認對話方塊的標題與啟動它的命令相同。  
  
-   確認所有的標準控制項有一致的作業系統: 背景色彩是標準，而且沒有控制項應該具有特殊的 re 樣板樣式，讓它們顯示不同於標準的控制項。  
  
-   確認在表單中的邊界應該 12 個像素，而且應該會出現統一且一致。  
  
-   確認對話方塊出現在 IDE 介面或繁衍 \(spawn\) 它們的視窗置中。  
  
-   時很有用，對話方塊應該是可調整大小。 可調整大小的對話，您可以確認，根據調整大小，適當的控制項必須調整大小\] 對話方塊的其他部分保持不變時。  
  
-   確認可調整大小的對話保存任何使用者調整的大小 \(大小、 位置、 擴充的對話方塊控制項等\)。  
  
-   請確認在標題列中沒有圖示。  
  
-   請確認在標題列中沒有最大化按鈕。  
  
#### 對話方塊作業按鈕 \(VS 限用戶端\)  
  
-   確認作業按鈕會依此順序: **確定**, ，**取消**, ，**套用**。  
  
-   確認 **確定** 和 **取消** 按鈕是標準的大小: 75 x 23 像素為單位。  
  
-   確認 **確定** 和 **取消** 按鈕都屬於相同的大小，不論字串長度。  
  
-   如果上一個作業按鈕標籤需要比標準寬度\] 按鈕，確認對應 **取消** 按鈕是相等的大小。  
  
-   確認有 6 個像素之間填補量按鈕相關聯的控制項。  
  
-   確認 **確定** 和 **取消** 按鈕不會有助憶鍵 \(便捷鍵加底線的字母所定義\)。  
  
-   請確認該一個按鈕 \(通常 **確定**\) 預設有焦點。  
  
-   確認 **Esc** 取消對話方塊  
  
-   確認 **Enter** 如果焦點不會處理輸入控制項中執行的預設按鈕。  
  
-   確認 **確定** 和 **取消** 按鈕會定位在右下角的 \[\] 對話方塊中。 在極少數的例外狀況是接受垂直堆疊右上角。  
  
-   確認只有當其他按鈕的對話方塊內的水平對齊方式，就會使用垂直的組態。  
  
### 控制標準  
  
#### 一般  
  
-   確認，如果可能的話，有很好的預設值，以加快使用者互動，將使用者導向到安全或常見的結果。  
  
-   請確認該標準控制項的行為方式，讓使用者知道會發生什麼情況根據之前的經驗。  
  
#### Label 控制項  
  
-   確認每個控制項都有標籤，而且每個標籤以視覺化方式搭配其控制項 \(通常是在 4\-6 像素範圍\)，並更接近其對應的控制項比其他控制項。  
  
-   請確認標籤位於清除與控制項左邊緣定位上方如果左右，讓標籤的基準線對齊輸入文字的基準線如果位於左邊水平置中。  
  
-   確認是否左邊的控制項定位的數個堆疊的標籤和輸入的控制項，標籤靠左對齊，且等於距離的對話方塊中，永遠不會排清右方和垂直尺規的控制項。 除非它們需要額外的間距，以指出群組應該平均分散組。  
  
#### 輸入的控制項 \(文字方塊和下拉式方塊\)  
  
-   確認使用時的預設環境字型，文字方塊、 下拉式方塊和按鈕的高度顯示所有 23 像素為單位。  
  
-   如果提示文字，請確認色彩設為 `Environment.ControlEditHintText` 使用色彩服務。  
  
-   如果欄位是必要的欄位，因此必須加以識別，請確認:  
  
    -   設為背景 `Environment.ControlEditRequiredBackground` 且設為前景 `Environment.ControlEditRequiredHintText`  
  
    -   會顯示為控制項中的提示文字 **"\< Required \>"**  
  
#### 按鈕控制項  
  
-   確認按鈕的最小大小為 75 x 23 像素，除非容納較長的文字。  
  
-   確認按鈕已離開，和右邊緣 3\-5 像素為單位，以及內容的邊框間距。  
  
-   它使用是可接受的小方形按鈕，具有省略符號 **\[...\]** 上，而不是 **\[瀏覽...\]** 按鈕 \(或類似的功能\)。 如果使用，請確認按鈕為 23 x 23 的大小。  
  
-   如果有多個 **\[瀏覽...\]** 按鈕在對話方塊中，然後確認精簡的版本 \(僅限省略符號 **\[...\]**\) 用於所有。  
  
-   請確認該省略 **\[...\]** 按鈕不會有助憶鍵。 焦點旁邊輸入控制項上時，有一個索引標籤應該將焦點移至省略符號按鈕。  
  
-   確認按鈕、 命令和命令連結來啟動第二個 UI，來擷取更多的使用者輸入必須以省略符號結尾 **\[...\]**。  
  
#### 超連結  
  
-   請確認超連結控制項永遠不會閃爍紅時作用中。 這是色彩服務未使用指標  
  
-   確認所使用的 VS 色彩:  
  
    -   `Environment.ControlLinkText`  
  
    -   `Environment.ControlLinkTextHover`  
  
    -   `Environment.ControlLinkTextPressed`  
  
-   確認出現藍色沒有底線除非段落中內嵌超連結。  
  
#### 核取方塊  
  
-   如果核取方塊具有多行文字，請確認方塊對齊第一行的文字，不垂直置中跨所有行。  
  
-   確認核取方塊永遠指出二進位選項和執行不使用者瀏覽或開啟新視窗或頁面。  
  
-   如果核取方塊會顯示輸入控制項的相關選項，請確認位於靠左和很接近下該控制項，以指出其關聯性。  
  
-   確認已核取方塊 **從未** 做為用來啟用對話方塊或頁面的整個內容。  
  
#### 群組方塊  
  
-   確認\] 對話方塊不包含單一群組內的方塊，它包含的全部內容\] 對話方塊。  
  
-   確認至少兩個控制項內的每一個群組方塊。  
  
-   不應該有兩個以上的群組方塊\] 對話方塊上。  
  
-   確認沒有任何巢狀的群組方塊。  
  
### 圖示  
  
-   確認在暗色調佈景主題中的圖示出現正確反向。  
  
-   請確認所有圖示為都基礎的核心概念。  
  
-   確認每個圖示不同，輕易地識別，且不包含兩個以上的概念 \(不含狀態修飾詞\/語言\)。  
  
-   確認基底的圖示出現在空間中置中。  
  
-   確認所有圖示都出現在高對比模式易於閱讀。  
  
-   確認使用任何色彩配合色彩使用量標準。  
  
-   確認沒有任何暈圈 \(框線\) 周圍的圖示。 \(如果有的話，halo 應該符合相鄰 UI 的背景色彩\)。  
  
### 觸控式 UI  
  
-   確認互動式控制項都大到足以具有輕鬆 touchable – 最小 **23 x 23 像素** 的大小  
  
-   請確認至少會最常使用的控制項 **40 x 40 像素** 的大小。  
  
-   請確認至少有互動式控制項 **5 像素的間距大小** 兩者之間
